# План розгортання Wolf Pack

## Варіанти хостингу

### 1. Render.com (безкоштовно)
**Переваги:**
- Безкоштовний план для бекенду (750 годин/місяць)
- Безкоштовна PostgreSQL база (90 MB)
- Автоматичне розгортання з GitHub
- SSL сертифікати автоматично
- Масштабування при зростанні

**Недоліки:**
- Обмеження по ресурсах
- Сплячі сервіси після 15 хвилин без активності (прокидаються при запиті)
- Database обмеження 90 MB

**Вартість:** $0/місяць

### 2. VPS (DigitalOcean/Hetzner) (~$5/місяць)
**Переваги:**
- Повний контроль
- Необмежені ресурси
- 24/7 робота
- Можливість розгорнути кілька проектів

**Недоліки:**
- Потрібно налаштовувати вручну
- Потрібен домен (додаткові витрати)
- SSL сертифікати потрібно налаштовувати

**Вартість:** $5-10/місяць

## Кроки розгортання на Render.com

### 1. Підготовка репозиторію GitHub
- Створити репозиторій для гри
- Завантажити код:
  - `backend/` — бекенд API
  - `frontend/` — статичні файли гри (HTML, JS, CSS)
  - `database/` — міграції
  - `bot/` — Telegram бот (окремий сервіс)

### 2. Налаштування Render.com
- Створити акаунт на Render.com
- Підключити GitHub репозиторій
- Створити Web Service для бекенду:
  - **Build Command:** `cd backend && npm install`
  - **Start Command:** `node server.js`
  - **Environment Variables:**
    - `PORT`: 10000 (Render автоматично призначає)
    - `DATABASE_URL`: з Render PostgreSQL
    - `TELEGRAM_BOT_TOKEN`: токен бота
    - `NODE_ENV`: production

### 3. PostgreSQL Database на Render
- Створити PostgreSQL базу
- Отримати `DATABASE_URL`
- Запустити міграції: `psql $DATABASE_URL -f database/001_initial.sql`

### 4. Frontend на GitHub Pages
- Використати GitHub Pages для фронтенду
- Налаштувати Custom Domain (опціонально)
- Оновити URL бекенду у фронтенді

### 5. Telegram Bot Webhook
- Налаштувати вебхук для бота на URL Render.com
- Використати `setWebhook` API Telegram

### 6. Тестування
- Перевірити API: `https://your-service.onrender.com/api/health`
- Перевірити фронтенд: `https://your-username.github.io/wolf-eggs-game`
- Перевірити бота: відправити `/start`
